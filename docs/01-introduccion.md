# Introducción

## Tema
Este trabajo aborda **CI/CD con GitHub Actions** aplicado a un proyecto **Node + TypeScript** que se despliega automáticamente a **GitHub Pages**.

## Contexto en Ingeniería de Software
El tema se ubica principalmente en:
- **Build & Release Engineering**
- **DevOps**
- **Aseguramiento de calidad** (Quality Assurance)
- **Gestión de configuración** (SCM) y control de cambios

## Importancia
CI/CD reduce riesgos de integración, mejora la calidad y habilita entregas frecuentes y confiables.

## Objetivo de este repositorio
Construir un cuerpo de conocimiento **reproducible**: el lector debe poder clonar el repo, ejecutar comandos y comprobar que:
- CI valida calidad y ejecuta tests.
- Se generan artifacts (coverage, build).
- Se ejecutan controles de seguridad.
- Se despliega un sitio estático a Pages cuando CI pasa.
